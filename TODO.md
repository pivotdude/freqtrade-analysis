# План разработки дополнительной торговой статистики

Этот документ описывает план добавления расширенных метрик для анализа торговой истории Freqtrade.

## Общий подход

1.  - [ ] **Модификация `DatabaseService`**: Добавить новые методы для извлечения данных, необходимых для сложных расчетов (например, всех сделок для построения кривой эквити).
2.  - [ ] **Расширение `TradeAnalyzer`**: Реализовать новые методы для расчета каждой метрики.
3.  - [ ] **Обновление `trade.types.ts`**: Добавить новые интерфейсы для хранения рассчитанных статистических данных.
4.  - [x] **Интеграция в отчет**: Вывести новые метрики в консоль (`index.ts`) и/или в Markdown отчет (`MarkdownReportGenerator.ts`).

## 1. Максимальная просадка (Max Drawdown)

**Задача:** - [x] Рассчитать максимальное снижение эквити (баланс + стоимость открытых позиций) за весь период.

**Данные для расчета:**
*   Все сделки (`trades`) с датами открытия/закрытия и суммами.
*   Нужен начальный капитал (можно задать как константу для начала).

**Где брать данные:**
*   Метод в `DatabaseService`, который вернет все сделки: `getAllTrades()`.

**План реализации:**
1.  Получить все сделки, отсортированные по дате открытия.
2.  Создать временную шкалу событий (сделка открыта, сделка закрыта).
3.  Симулировать изменение баланса во времени:
    *   Начать с `initialCapital`.
    *   При закрытии сделки, `balance += net_profit`.
    *   Для каждой точки на временной шкале (например, каждый час или каждый день), эквити = `balance + unrealized_pnl`.
    *   **Проблема:** Расчет `unrealized_pnl` требует исторических данных о ценах, которых нет в БД.
    *   **Упрощение (v1):** Рассчитывать просадку только по закрытым сделкам (drawdown по балансу, а не по эквити). Это менее точно, но реализуемо с текущими данными.
4.  В процессе симуляции отслеживать `peak_equity` и `max_drawdown`.
    *   `peak_equity = max(peak_equity, current_equity)`
    *   `drawdown = (peak_equity - current_equity) / peak_equity`
    *   `max_drawdown = max(max_drawdown, drawdown)`

## 2. Profit Factor

**Задача:** - [ ] Рассчитать отношение общей прибыли к общему убытку.

**Данные для расчета:**
*   `close_profit_abs` для каждой сделки.

**Где брать данные:**
*   Данные уже доступны в `TradeAnalyzer`.

**План реализации (в `TradeAnalyzer`):**
1.  `gross_profit = sum(close_profit_abs)` для всех сделок, где `close_profit_abs > 0`.
2.  `gross_loss = sum(abs(close_profit_abs))` для всех сделок, где `close_profit_abs < 0`.
3.  `profit_factor = gross_profit / gross_loss`. Обработать случай, когда `gross_loss = 0`.

## 3. Expectancy (Математическое ожидание)

**Задача:** - [ ] Рассчитать среднюю ожидаемую прибыль на одну сделку.

**Данные для расчета:**
*   Win Rate, Loss Rate, Avg Win, Avg Loss.

**Где брать данные:**
*   Рассчитывается на основе всех сделок в `TradeAnalyzer`.

**План реализации (в `TradeAnalyzer`):**
1.  `win_rate = profitable_trades / total_trades`.
2.  `loss_rate = losing_trades / total_trades`.
3.  `avg_win = sum(net_profit)` для прибыльных сделок / `profitable_trades`.
4.  `avg_loss = abs(sum(net_profit))` для убыточных сделок / `losing_trades`.
5.  `expectancy = (win_rate * avg_win) - (loss_rate * avg_loss)`.

## 4. Коэффициенты Sharpe / Sortino

**Задача:** - [ ] Оценить доходность с поправкой на риск.

**Данные для расчета:**
*   Временной ряд доходности портфеля (например, ежедневные P/L).
*   Безрисковая ставка (можно принять за 0).

**Где брать данные:**
*   Требует той же симуляции эквити, что и для Max Drawdown.

**План реализации:**
1.  В рамках симуляции для Max Drawdown, создать массив доходностей за каждый период (день/час): `returns.push(period_pnl / equity_at_period_start)`.
2.  Рассчитать `avg_return` и `std_dev_return` (стандартное отклонение доходностей).
3.  `sharpe_ratio = (avg_return - risk_free_rate) / std_dev_return`.
4.  Для Sortino: рассчитать `downside_std_dev` (стандартное отклонение только для отрицательных доходностей).
5.  `sortino_ratio = (avg_return - risk_free_rate) / downside_std_dev`.
*   **Примечание:** Это сложный расчет, отложить после реализации просадки.

## 5. Максимум одновременно открытых позиций (Exposure)

**Задача:** - [ ] Найти максимальное количество одновременно открытых сделок и максимальную сумму задействованного капитала.

**Данные для расчета:**
*   `open_date`, `close_date`, `stake_amount` для всех сделок.

**Где брать данные:**
*   `DatabaseService.getAllTrades()`.

**План реализации:**
1.  Создать плоский список всех событий: `[{date: open_date, type: 'open', stake: stake_amount}, {date: close_date, type: 'close', stake: stake_amount}]`.
2.  Отсортировать события по дате.
3.  Итерировать по списку, поддерживая счетчики `current_open_trades` и `current_exposure_amount`.
    *   При 'open': `current_open_trades++`, `current_exposure_amount += stake`.
    *   При 'close': `current_open_trades--`, `current_exposure_amount -= stake`.
4.  На каждом шаге обновлять `max_open_trades = max(max_open_trades, current_open_trades)` и `max_exposure_amount = max(max_exposure_amount, current_exposure_amount)`.

## 6. Slippage (Проскальзывание)

**Задача:** - [ ] Рассчитать разницу между запрашиваемой и исполненной ценой ордера.

**Данные для расчета:**
*   Данные из таблицы `orders`. Нужны `ft_price` (цена запроса) и `average` (средняя цена исполнения).

**Где брать данные:**
*   `DatabaseService` должен быть расширен, чтобы получать сделки вместе со связанными с ними ордерами. Тип `Trade` уже имеет опциональное поле `orders?: Order[]`.

**План реализации:**
1.  В `DatabaseService`, при загрузке сделки, также загружать связанные ордера.
2.  В `TradeAnalyzer`, для каждой сделки и ее ордеров на покупку:
    *   `slippage = (average_price - ft_price) / ft_price * 100`.
3.  Рассчитать среднее проскальзывание по всем сделкам.

## 8. Эффективность сделок (Профит в час)

**Задача:** - [ ] Для каждой сделки рассчитать длительность и доходность в пересчете на час.

**Данные для расчета:**
*   `open_date`, `close_date`, `close_profit_pct`.

**Где брать данные:**
*   Данные уже есть в объекте `Trade`.

**План реализации:**
1.  При анализе каждой сделки в `TradeAnalyzer` или при форматировании в `MarkdownReportGenerator` добавить расчет:
    *   `duration_minutes = (new Date(close_date) - new Date(open_date)) / 60000`.
    *   `profit_per_hour_pct = (close_profit_pct / duration_minutes) * 60`.
2.  Можно выводить как для каждой сделки, так и среднее значение.

## 9. Комиссия в % от профита

**Задача:** - [ ] Показать, какую долю от валовой прибыли "съедает" комиссия.

**Данные для расчета:**
*   `fee_open_cost`, `fee_close_cost`, `close_profit_abs`.

**Где брать данные:**
*   Данные уже есть в объекте `Trade`. Валовая прибыль (`gross_profit`) это `close_profit_abs` (прибыль до вычета комиссии).

**План реализации:**
1.  Для каждой сделки:
    *   `total_fee = fee_open_cost + fee_close_cost`.
    *   `gross_profit = close_profit_abs`.
    *   `fee_as_pct_of_profit = (total_fee / gross_profit) * 100`. (Обработать `gross_profit <= 0`).
2.  Добавить эту информацию в таблицу с деталями сделок в отчете.
