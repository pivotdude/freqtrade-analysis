# Security Policy

## Supported versions

This project is currently in active early development.
Security fixes are prioritized for the latest commit on the default branch.

## Reporting a vulnerability

If you discover a potential security vulnerability, please report it privately.

- Preferred channel: open a private security advisory on GitHub (if available for this repo).
- Fallback channel: open an issue with minimal details and request a private contact path.

Please include:

- a clear description of the vulnerability
- impact assessment
- reproduction steps or proof of concept
- suggested remediation (if known)

## Response process

Maintainers aim to:

1. Acknowledge receipt within **3 business days**.
2. Confirm severity and impact.
3. Prepare and release a fix as soon as practical.
4. Credit the reporter (if desired) after disclosure.

## Disclosure policy

Please avoid public disclosure until a fix is available or coordinated disclosure is agreed.
